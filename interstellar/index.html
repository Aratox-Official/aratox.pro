Self-Hosting Interstellar on GitHub Codespaces

This is guide of how to get Interstellar (a web proxy) running on GitHub Codespaces for free, using the ad-free branch. You don't need anything other than a GitHub account.

GitHub gives you 120 core-hours and 15 GB-months of Codespace usage per month on the free plan, which is more than enough for this.

Go to github.com/codespaces and click New codespace in the top right. From there, click Select a repository and type in UseInterstellar/Interstellar. Select it, then open the Branch dropdown and switch it from main to Ad-Free. Leave the machine type at the default 2-core and hit Create codespace.

This should take a minute to set up your codespace. Once the browser-based editor loads, you're in.

Using the Ad-Free branch means ads are stripped out of the UI. The proxy itself works the same as the main branch.

Open the terminal with Ctrl + ` if it isn't already showing, then run:

npm install

Wait for it to finish - you'll see something like added N packages when it's done. Then start the server:

npm start

The server runs on port 8080 by default. You'll see Listening on port 8080 in the terminal output.

Click the Ports tab in the bottom panel. Port 8080 should be listed with a forwarded address. Hover over it and click the globe icon to open it in a new tab.

The link will look something like https://<your-codespace-name>-8080.app.github.dev. By default, the port is private - so only you can access it while signed into GitHub. If you want to share it, right-click the port entry and set visibility to Public - and you'll get a popup to confirm this when you first open it anyway, just click Make public.

If the proxy isn't loading correctly, double-check that you clicked Make public. This is the most common reason it breaks.

You have successfully set up Interstellar and are ready to browse the web freely. Some websites may not work as a result of the way that Interstellar's tunnelling works, but most websites are supported.

The following steps will explore how to turn off the proxy (once you're done using it; this'll save compute resources). They are not related to the setup of Interstellar.

To stop the proxy without shutting down the whole Codespace, press Ctrl + C in the terminal. That's it - the Node process stops and you're back at a normal shell. Run npm start again whenever you want to bring it back up.

If the process is frozen and Ctrl + C isn't doing anything, you can force-kill it:

pkill -f "node"

When you're done using the proxy for the day, stop the Codespace so it doesn't use your free compute hours. Go back to github.com/codespaces, click the ⋯ menu next to your Codespace, and hit Stop codespace.

Codespaces auto-suspend after 30 minutes of inactivity anyway, but stopping manually is cleaner. A stopped Codespace only counts against your storage quota, not your compute hours. Next time you come back, just resume it from the same dashboard (github.com/codespaces) and run npm start again.

If you want to get rid of the codespace entirely, you can delete it from the dashboard. Just make sure you don't have any uncommitted changes you care about - deleting a Codespace is permanent.

Quick reference

# Install dependencies
npm install

# Start the server
npm start

# Stop the server
Ctrl + C

# Force-kill if frozen
pkill -f "node"
